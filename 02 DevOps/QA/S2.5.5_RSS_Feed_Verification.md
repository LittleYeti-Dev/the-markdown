## Issue 1: Pretty Permalink Not Working

**Snippet ID 74** (S2-G5-M15 RSS Feed Generator) registers the feed via:
```php
add_feed( 'the-markdown', 'ns_render_rss_feed' );
```

This creates a rewrite rule for `/feed/the-markdown`. However, on WordPress.com, rewrite rules require a flush (typically via Settings > Permalinks save) to take effect. The feed works via query parameter `/?feed=the-markdown` which bypasses rewrite rules.

**Impact:** Low — feed is accessible, just not at the documented pretty URL.
**Fix:** Save permalinks in WordPress admin (Settings > Permalinks > Save Changes) to flush rewrite rules. No code change needed.

## Issue 2: Zero Items in Custom Feed

The custom feed queries for promoted feed items but returns zero. This is consistent with:
- All 760 feed items have `promote_status: "dev"` and `block_assignment: 0`
- No editorial promotion workflow has been executed
- The feed generator likely filters for promoted items only

**Impact:** Expected operational state. Not a code defect. Feed will populate once items are promoted through the editorial workflow.

**Verification via CPT feed:** The standard WordPress CPT feed at `/feed/?post_type=ns_feed_item` returns all 760 items with proper RSS 2.0 formatting, confirming the feed item data is accessible via RSS.

## Issue 3: Feed Not Discoverable in HTML

Page HTML only contains standard WordPress feed links:
```html
<link rel="alternate" type="application/rss+xml" title="Justin Kuiper » Feed" href="https://justin-kuiper.com/feed/" />
<link rel="alternate" type="application/rss+xml" title="Justin Kuiper » Comments Feed" href="https://justin-kuiper.com/comments/feed/" />
```

The custom "The Markdown" feed is **not** included in the HTML `<head>` for auto-discovery.

**Impact:** Medium — RSS readers that rely on auto-discovery won't find the custom feed.
**Fix:** Add a `<link rel="alternate">` tag for the custom feed via `wp_head` action. Requires a small snippet addition (Sprint 3 candidate).
