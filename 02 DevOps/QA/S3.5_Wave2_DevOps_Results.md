**Why `/the-markdown-2/`?** The slug `the-markdown` is reserved by `add_feed('the-markdown', ...)` in snippet #74 (RSS Feed Generator). WordPress refuses to assign an existing feed name as a page slug. A `template_redirect` hook performs the 301 redirect from the canonical URL.

## Issues Encountered

1. **Slug conflict:** `the-markdown` slug reserved by `add_feed()` in snippet #74. Resolved with `/the-markdown-2/` page + 301 redirect snippet.

2. **CSS conditional mismatch:** Initial CSS used `is_page('the-markdown')` which didn't match the actual slug. Fixed by switching to `is_page(1077)` (page ID).

3. **domain_tag vs content_domain:** The prompt assumed `tax_query` with `content_domain` taxonomy, but items are actually classified via `domain_tag` post meta. Adjusted to use `meta_query` instead.

4. **Domain slug differences:** Actual taxonomy term slugs (`ai-machine-learning`, `cybersecurity`, etc.) differ from the short `domain_tag` meta values (`ai`, `cyber`, etc.). Data query uses the meta values correctly.

## Recommendations for Phase 2 (Wave 3 / Sprint 4)

1. **Commentary content:** Populate `jk_commentary` post meta on feed items so commentary cards show real editorial perspective instead of placeholder text.

2. **Featured images:** Integrate Canva-generated card images as featured media on feed items to replace the hero placeholder box.

3. **Slug resolution:** Consider renaming the RSS feed from `the-markdown` to `the-markdown-feed` in snippet #74, which would free the `the-markdown` slug for the page and eliminate the redirect chain.

4. **Domain nav links:** Wire domain nav items to filtered archive views (e.g., `/archive/?domain=ai`) instead of `#` placeholders.

5. **Edition banner:** Pull from a custom option or post meta so it can be updated editorially without code changes.

6. **PUSH button integration:** Connect Instagram, Medium, YouTube share URLs to real platform endpoints.

7. **"NEXT REFRESH" timer:** Make dynamic with a countdown to the next cron ingestion run.

8. **WP theme full-width template:** Consider applying a full-width page template to maximize content area width on desktop viewports.
