DR-0028: Nav Link Fix — Direct Edit vs Snippet Filter

Decision: Fix the nav URL directly in WP Admin menu editor rather than deploying a wp_nav_menu_objects filter snippet.

Rationale: The broken URL was a one-time data error (wrong subdomain in menu item), not a systemic issue requiring a runtime filter. Direct edit is simpler, has no performance overhead, and doesn't add to the 120+ snippet inventory. The API was also returning 401 at the time, making direct WP Admin the fastest path to resolution.

The h3ndriksj account is the actual admin (user ID 204791015) and successfully authenticates against both core and Code Snippets endpoints. Operational Playbook needs updating.
